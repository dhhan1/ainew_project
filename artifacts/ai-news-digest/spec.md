# AI News Trend 비서 — Specification

## 개요

매일 아침 06:00 KST에 5~6개 신뢰 RSS 소스(EN/KR 혼합)에서 직전 24시간 기사를 모아 교차 출처 + 최신순 기준으로 Top 10을 뽑고, 한국어 요약을 단일 페이지에 카테고리별 박스로 그룹핑해 보여주는 비서. 사용자는 13개 사이트를 일일이 방문하지 않아도 그날의 핵심 AI 뉴스를 한 페이지에서 파악한다.

## 범위

### 포함
- `/` — 카테고리 박스로 그룹핑된 Top 10 카드 리스트, 마지막 갱신 시각(KST), 수동 새로고침 버튼, **다크/라이트 모드 토글**, **그날의 분위기 배지**(이모지 + 한국어 라벨)
- `/news/[id]` — 한국어 요약 전문, 원문 링크 버튼, 같은 클러스터 내 다른 기사 링크, 메타(출처·발행 시각·수집 시각)
- 매일 06:00 KST 자동 갱신 (Vercel Cron)
- 같은 사건을 여러 매체가 보도하는 경우 클러스터로 묶고 "N개 매체 보도" 배지 표시
- 5개 사전 정의 카테고리(모델/기업, 연구, 규제·정책, 응용, 기타)로 박스 그룹핑
- 한국어 자동 요약 (Azure OpenAI gpt-5.4 deployment)
- 일부 소스 실패 시 graceful degradation (정상 소스로만 결과 채움)
- 기사별 요약 실패 시 1회 재시도 후 fallback 표시
- **다크 모드** — 사용자가 토글로 라이트/다크 전환. 선택은 브라우저에 영구 저장(`localStorage`).
- **그날의 분위기 배지** — 페이지 제목 옆에 이모지 + 한국어 라벨로 그날의 트렌드 강도를 한눈에 보여준다.

### 제외
- **사용자 인증/로그인** — MVP 1차에서는 공개 페이지. 개인화는 다음 사이클에서 검토.
- **검색·필터·태그 UI** — Top 10이 좁은 결과라 필터 가치가 낮음. 카테고리 박스 그룹핑이 그 역할을 일부 대체.
- **이메일·푸시 다이제스트** — 외부 발송 서비스 셋업 비용. 다음 사이클로.
- **이력 보관·아카이브 페이지** — 매일 갱신하는 "오늘의 트렌드" 한 화면이 본질. DB 도입 회피.
- **댓글·공유·북마크** — engagement 기능은 MVP 외.
- **동적 카테고리** — gpt-5.4가 자유 라벨을 만들어내지 않고, 5개 사전 정의 셋 중 하나로만 분류한다.
- **수동 강제 캐시 무효화 UI** — 새로고침 버튼은 단순 페이지 리로드. 다음 cron 시각까지는 같은 데이터를 본다.
- **시스템 다크 모드 자동 추적(`prefers-color-scheme`)** — MVP에서는 사용자 토글만. OS 자동 추적은 다음 사이클로.

## 시나리오

### 1. 일일 갱신된 Top 10을 본다 (happy path)

- **Given** — 06:00 KST cron이 정상 완료되어 직전 24시간 기사가 카테고리별로 정리된 상태
- **When** — 사용자가 06:30 KST에 `/`에 접속한다
- **Then** — 5개 카테고리 박스 중 결과가 있는 박스만 화면에 나타나고, 박스 안에 카드가 들어 있으며, 페이지 상단에 "마지막 갱신: 2026-04-30 06:00 KST" 형태의 텍스트가 보인다

성공 기준:
- [ ] 페이지 상단에 "마지막 갱신: YYYY-MM-DD HH:mm KST" 형식 텍스트가 정확히 1회 보인다
- [ ] 화면에 보이는 모든 카드의 합이 10개 이하이다 (정확히 10일 수도, 24h 안에 기사가 부족하면 그 이하)
- [ ] 비어 있는 카테고리 박스는 화면에 렌더되지 않는다
- [ ] 모든 카드의 제목·요약·발행 시각·출처 라벨이 한국어 또는 한국어 기반 표기(원문 매체명은 영문 허용)로 보인다

### 2. 같은 사건을 여러 매체가 보도하면 한 카드로 묶인다

- **Given** — 같은 24시간 안에 3개 매체가 같은 사건(예: 신규 AI 모델 발표)을 각자 기사로 발행한 상태
- **When** — 사용자가 `/`에 접속해 그 사건이 포함된 카테고리 박스를 본다
- **Then** — 그 사건은 카드 1개로만 표시되고, 카드 안에 "3개 매체 보도" 배지가 보인다. 대표로 표시되는 제목·요약은 클러스터 내 점수 최고 기사의 것이다

성공 기준:
- [ ] 같은 사건을 다룬 기사 3건이 입력되면 화면에 카드는 정확히 1개 노출된다
- [ ] 그 카드 안에 "3개 매체 보도" 텍스트(또는 동등한 한국어 표현)의 배지가 보인다
- [ ] 카드의 제목·요약은 클러스터 내 점수 최고 기사의 한국어 요약이다 (동점 시 더 최신 기사)

### 3. 카드를 클릭하면 상세 페이지로 이동해 한국어 전문과 원문 링크를 본다

- **Given** — `/`에 카테고리 박스 안에 카드가 표시된 상태
- **When** — 사용자가 카드 본문 영역을 클릭한다
- **Then** — `/news/[id]` 페이지로 이동하고, 그 페이지에 한국어 요약 전문(3~5줄), "원문 보기" 버튼, 같은 클러스터의 다른 기사 링크 목록(있다면), 메타(출처·발행 시각·수집 시각)가 표시된다. "원문 보기" 버튼을 클릭하면 새 탭에서 해당 매체의 원문 URL이 열린다

성공 기준:
- [ ] 카드 본문 클릭 시 URL이 `/news/<id>` 패턴으로 변한다
- [ ] 상세 페이지에 한국어 요약 텍스트가 3~5줄 분량으로 보인다
- [ ] 상세 페이지에 "원문 보기" 또는 동등한 한국어 라벨의 링크/버튼이 보이고, 그 link target은 새 탭(`target="_blank"` 동등)으로 열린다
- [ ] 클러스터 크기가 2 이상이면 "관련 기사" 섹션이 나타나고 다른 기사들의 매체명·제목·원문 링크가 보인다
- [ ] 클러스터 크기가 1이면 "관련 기사" 섹션은 보이지 않는다
- [ ] 메타 영역에 출처 매체명, 발행 시각(KST), 수집 시각(KST)이 보인다

### 4. Top 10이 사전 정의 카테고리 박스로 그룹핑된다

- **Given** — Top 10 카드가 5개 카테고리(모델/기업, 연구, 규제·정책, 응용, 기타)에 분포한 상태
- **When** — 사용자가 `/`에 접속한다
- **Then** — 카드들이 카테고리 박스 안에 모여 있고, 각 박스 상단에 카테고리 이름이 라벨로 표기된다. 박스의 표시 순서는 박스 내부 최고 점수 기준 내림차순이다. 박스 내 카드 순서도 점수 내림차순이다

성공 기준:
- [ ] 화면에 보이는 박스의 라벨은 정의된 5개 카테고리명 중 하나에 해당한다
- [ ] 같은 박스 안의 카드 수가 1개 이상일 때만 박스가 렌더된다 (0개면 박스 자체 안 보임)
- [ ] 박스 안의 카드 순서는 점수 내림차순이며, 동점 시 더 최신 기사가 위로 온다
- [ ] 박스의 화면 순서는 박스 내 최고 점수 카드의 점수 기준 내림차순이다

### 5. 일부 RSS 소스가 실패해도 나머지로 결과를 채운다

- **Given** — 5~6개 소스 중 1~2개가 응답 실패(timeout, 4xx, 5xx, 네트워크 오류)인 상태
- **When** — 06:00 KST cron이 실행된다
- **Then** — 정상 응답한 소스의 기사만으로 클러스터링·점수·요약을 수행해 결과 페이지를 만든다. 사용자가 `/`에 접속하면 부분 결과를 정상 표시하고, 실패 사실은 페이지에 별도 노출하지 않는다 (서버 로그에만 기록)

성공 기준:
- [ ] 6개 소스 중 임의 1개를 실패시켰을 때 페이지가 에러 화면 없이 렌더된다
- [ ] 그 페이지의 카드 합이 1개 이상이다 (정상 소스 기사 0건 케이스가 아닌 한)
- [ ] 페이지 어디에도 "오류", "실패" 같은 사용자 노출 메시지가 없다
- [ ] 실패한 소스 이름과 사유가 서버 로그(stdout/stderr)에 1줄 이상 기록된다

### 6. 기사별 요약이 실패하면 1회 재시도 후 fallback

- **Given** — Top 10 안의 한 기사에 대해 Azure OpenAI 호출이 일시 실패(예: 429 rate-limit, 5xx)인 상태
- **When** — 06:00 KST cron이 그 기사의 한국어 요약을 만드는 단계에서 첫 호출이 실패한다
- **Then** — 같은 호출을 1회 재시도하고, 재시도까지 실패하면 그 기사 카드의 요약 자리에 RSS의 원문 description(있다면 그대로) 또는 "요약을 가져오지 못했습니다" 텍스트를 보여준다. 카드 자체는 페이지에 그대로 노출된다

성공 기준:
- [ ] 한 기사의 요약 호출이 1회 실패 후 재시도에서 성공하면 그 기사 카드에 정상 한국어 요약이 보인다
- [ ] 재시도까지 실패하면 그 카드의 요약 자리에 RSS 원문 description 또는 "요약을 가져오지 못했습니다" 한국어 fallback 텍스트가 보인다
- [ ] 다른 9개 기사의 카드는 영향받지 않고 정상 한국어 요약을 보여준다

### 7. 그날의 분위기를 페이지 제목 옆 배지로 한눈에 본다

- **Given** — 06:00 KST cron이 정상 완료되어 그날의 Top 카드와 클러스터 분포가 결정된 상태
- **When** — 사용자가 `/`에 접속한다
- **Then** — 페이지 제목("AI News Trend") 옆에 그날의 분위기를 나타내는 배지가 정확히 1개 표시된다. 배지는 다음 3개 중 하나다 — 🌿 평온한 날 / ☀️ 보통의 날 / 🔥 주목 필요한 날. 분위기 결정 규칙:
  - 🌿 **평온한 날** — 그날 화면에 보이는 카드의 총 개수가 5건 이하 (24h 내 새 기사 부족)
  - 🔥 **주목 필요한 날** — Top 안에 클러스터 크기 ≥ 3인 카드(=3개 이상 매체가 같은 사건 보도)가 1개 이상
  - ☀️ **보통의 날** — 위 두 조건에 해당하지 않는 모든 경우 (기본 상태)

성공 기준:
- [ ] 카드 총 개수가 5 이하면 제목 옆 배지에 "🌿 평온한 날" 텍스트가 보인다
- [ ] 카드 중 클러스터 크기 ≥ 3인 카드가 1개라도 있으면 "🔥 주목 필요한 날" 배지가 보인다 (위 평온 조건과 동시에 해당하면 주목 우선)
- [ ] 위 두 조건에 모두 해당하지 않으면 "☀️ 보통의 날" 배지가 보인다
- [ ] 배지는 페이지 어느 시점에서도 정확히 1개만 표시된다 (3개 중 동시 노출 금지)

### 8. 사용자가 라이트/다크 모드를 토글한다

- **Given** — `/`가 라이트 모드로 표시되어 있는 상태
- **When** — 사용자가 페이지 헤더의 모드 토글 버튼을 클릭한다
- **Then** — 페이지 전체가 다크 테마로 즉시 전환되고, 사용자가 페이지를 새로 열거나 다른 페이지(`/news/[id]`)로 이동했다가 돌아와도 다크 모드가 유지된다. 다시 토글하면 라이트로 돌아가고, 그 선택도 같은 방식으로 유지된다

성공 기준:
- [ ] 토글 버튼 클릭 후 `<html>` 또는 `<body>`에 다크 테마를 나타내는 클래스/속성이 적용된다 (예: `dark` 클래스 또는 `data-theme="dark"`)
- [ ] 다크 모드 상태에서 페이지 배경이 어두운 색이 되고, 텍스트는 밝은 색으로 보여 명도 대비가 라이트 모드와 반전된다
- [ ] 다크 모드 상태에서 `/news/[id]` 상세 페이지로 이동하면 그 페이지도 다크 테마로 표시된다
- [ ] 페이지를 새로고침해도 마지막 선택한 테마가 유지된다 (`localStorage` 같은 영구 저장 사용)

### 9. 사용자가 페이지를 즉시 다시 보고 싶을 때 새로고침한다

- **Given** — `/` 페이지가 열려 있고 마지막 갱신 시각이 06:00 KST인 상태
- **When** — 사용자가 페이지의 "새로고침" 버튼을 클릭한다
- **Then** — 페이지가 리로드되고, 마지막 갱신 시각은 여전히 06:00 KST(다음 cron 전이라면)인 채로 같은 데이터가 다시 보인다

성공 기준:
- [ ] 새로고침 버튼 클릭 후 페이지가 다시 로드된다 (URL 동일, 카드 다시 렌더됨)
- [ ] 클릭 후 보이는 마지막 갱신 시각은 직전 cron 시각과 같다 (수동 클릭으로 갱신이 트리거되지 않는다)

## 불변 규칙

| 범주 | 규칙 |
|---|---|
| **언어** | 사용자에게 노출되는 모든 텍스트(제목·요약·라벨·배지·버튼·에러 fallback)는 한국어 또는 한국어 기반 표기다. 매체 고유명·원문 URL은 영문 허용. |
| **시간대** | 사용자에게 보이는 모든 시각은 KST로 표시한다 (`YYYY-MM-DD HH:mm KST`). 사용자 로케일을 따라가지 않는다. |
| **데이터 신선도** | 화면에 보이는 카드는 마지막 갱신 시각 기준 직전 24시간 안에 발행된 기사만 포함한다. |
| **개인정보** | 외부 RSS만 소비하므로 사용자 개인정보를 수집·저장하지 않는다. 쿠키·트래킹 도입 금지. |
| **시크릿** | Azure 키와 cron 시크릿은 클라이언트 번들에 포함되지 않는다 (`NEXT_PUBLIC_` 접두사 절대 금지). |

## 의존성

- Azure OpenAI 리소스가 사전 프로비저닝되어 있어야 한다 (`gpt-5.4` chat deployment, `text-embedding-3-large` embedding deployment, API version `2024-12-01-preview`).
- Vercel 프로젝트가 생성되어 환경 변수 5종(`AZURE_API_KEY`, `AZURE_ENDPOINT`, `AZURE_DEPLOYMENT_NAME`, `AZURE_API_VERSION`, `AZURE_EMBEDDING_DEPLOYMENT`) + `CRON_SECRET`이 등록되어 있어야 한다.
- 6개 RSS 소스(또는 그 부분집합)가 외부에서 접근 가능해야 한다. 정확한 RSS URL은 plan/build 단계에서 라이브 검증 후 확정한다.

## 미결정 항목

- **6개 RSS 소스의 정확한 RSS URL** — Reuters Tech AI, MIT News AI, VentureBeat AI, artificialintelligence-news.com, AI Times(aitimes.com), AI Times KR(aitimes.kr) 후보 중 RSS 가용성·ToS·발행 빈도를 plan/build 단계에서 확인해 6개 또는 5개로 확정한다. 변경 비용 high — 후속 단계에서 반드시 결정.
